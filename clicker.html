Deuxième projet - Clone de Cookie Clicker

Dans le projet précédent, vous avez pu vous familiariser avec la conception de programme interactif. Votre créativité était toutefois assez limitée… En effet, les fonctions prompt et alert ne permettent d’interagir que par l’entremise de texte.

Il n’était même pas possible d’insérer des couleurs ou des images. Dans ce projet-ci par contre, vous pourrez donner libre cours à votre créativité. En effet, nous allons mettre de côté les boîtes de dialogues, pour plutôt nous attaquer à la création de pages web.

La page web que nous allons réaliser sera inspirée d’un jeu qui fut assez populaire : Cookie Clicker. Le principe de ce jeu est bien simple : cliquer le biscuit permet d’amasser d’autres biscuits et le but est d’amasser le plus de biscuits possible.

Malgré sa simplicité enfantine, ce jeu a connu un très grand succès au milieu de la décennie. Comme quoi une idée simple peut faire beaucoup de chemin. Imaginez simplement si ce projet portait un message, par exemple si chaque clic sur le biscuit équivalait à un don pour une association contre la famine.

L’impact que ça aurait eu est immense. Essayons donc de faire notre propre version de ce jeu, potentiellement avec un message. Pour ma part, je vais créer une page web qui permet de sensibiliser à la campagne du parti Québec Solidaire, tout en conservant le principe originel de Cookie Clicker. Tu peux cependant faire ta propre version en suivant le tutoriel. Laisse libre cours à ta créativité!

Une page web est structurée grâce au langage HTML. Pour créer une page web, il faut donc avoir un fichier HTML. Ce type de fichier peut être édité avec n’importe quel éditeur de texte. Tu peux même l’éditer en ligne (ici par exemple).

Un fichier HTML est structuré avec des balises. Les balises délimitent les sections de la page. Par exemple, une page commence avec la balise ouvrante <html> et se termine avec la balise </html>.

Un titre est balisé avec <h1> et </h1> (h1 est un titre, mais h2, h3 et ainsi de suite le sont aussi. h1 est simplement le plus gros).

Un paragraphe est balisé avec <p> et </p>.

Pour une image c’est un peu différent. Puisqu’elle ne peut pas contenir de texte, sa balise se ferme sur elle-même : <img />

Le code JavaScript quant à lui est balisé avec <script> et </script>.
