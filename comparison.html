Comparaisons

Explications

En JavaScript, les opérateurs de comparaison sont très similaires aux opérateurs de comparaisons mathématiques (>, ≥, <, ≤, =, ≠).

Comme en mathématiques, ces expressions peuvent être vraies ou fausses. Une comparaison retourne donc une valeur booléenne (true ou false). La seule différence est que certains opérateurs sont stricts et d’autres ne le sont pas.

Par exemple, il existe deux opérateurs différents pour exprimer l’égalité entre deux valeurs : == et ===. Le premier n’est pas strict, puisqu’il effectue une conversion de type. C’est-à-dire qu’il convertit les deux côtés de l’égalité au même type (nombre, chaîne de caractères, valeur booléenne...). De cette façon l’égalité 1 == "1" retourne true.

Par contre, l’opérateur === est strict. Il ne fait aucune conversion de type. L’égalité 1 === "1" retourne donc false.

De manière générale, il vaut mieux utiliser l’opérateur strict, puisqu’il est plus rapide et que la conversion de type peut parfois engendrer des comportements inattendus.

L’opérateur contraire à l’égalité est l’inégalité. En JavaScript, il se dénote != ou !==. Le deuxième étant la version stricte. Par exemple, 2 !== 3 retourne true. Il existe aussi, comme en mathématiques des opérateurs relatifs : <, >. Le premier (<) évalue si le terme de gauche est plus petit que celui de droite. Par exemple, 5 < 3 retourne false.

Le deuxième (>) évalue si le terme de gauche est plus grand que celui de droite. Par exemple, 5 > 3 retourne true.

Comme en mathématiques, il existe aussi les opérateurs plus petit ou égal et plus grand ou égal. Le premier se note <=. Le second se note >=.

Ils fonctionnent comme en mathématiques. Par exemple 3 <= 3 et 2 <= 3 retournent tous les deux true. Il en va de même pour 5 >= 5 et 6 >= 5.

Finalement, il existe aussi des opérateurs logiques. Ces opérateurs logiques retournent aussi une valeur booléenne (true ou false).

Il existe trois principaux opérateurs logiques : ||, && et !.

Le premier (||) est un OU logique. Si l’un des deux côtés de l’opérateur est true, il retourne true. Cependant, si les deux côtés sont false, il retourne false. En résumé :

Expression Résultat
true || true true
true || false true
false || true true
false || false false

Le deuxième (&&) est un ET logique. Il retourne true seulement si les deux côtés de l’expression sont true. Si l’un des deux côtés est false, il retourne false. En résumé :

Expression Résultat
true && true true
true && false false
false && true false
false && false false

Le troisième (!) est probablement le plus simple. Il s’agit d’un NON logique. Il ne compare pas vraiment, puisque contrairement aux autres opérateurs, il n’a pas de côté gauche. En effet, sa fonction se limite à retourner l’inverse de ce qui est à droite. En résumé :

Expression Résultat
!true false
!false true

Puisque tous ces opérateurs retournent des valeurs booléennes, il est tout à fait possible de les combiner, pour créer des expressions un peu plus complexes.1 Par exemple : (1 > 2) || (5 >= 5) retourne true.
