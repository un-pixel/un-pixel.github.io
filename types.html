Types de variables

Explications

Nous avons déjà vu, au chapitre précédent, qu’une variable peut contenir une valeur textuelle. C’est toutefois loin d’être le seul type de donnée que peut contenir une variable. Dans certains langages, la façon de déclarer une variable change en fonction de son type. Toutefois en JavaScript, la procédure reste la même. C’est une autre des raisons pour lesquelles ce langage est un bon point de départ.

Dans le cadre de ce tutoriel nous verrons que les variables peuvent contenir 6 différents types de données : une chaîne de caractères, un nombre, une valeur booléenne, une fonction, une liste ou un objet.

Il en existe toutefois plusieurs autres, qui ne seront pas abordés lors de ce cours.

Chaîne de caractères

Commençons avec quelque chose de familier : les données textuelles. De manière formelle elles sont nommées chaînes de caractères ou string en anglais. Elles sont généralement dénotées avec des guillemets anglais ("") ou des apostrophes (''). Par exemple : let nom = ‘John Doe’ ou let nom = “John Doe”.

Il existe aussi une troisième façon de dénoter les chaînes de caractère : avec le backtick (``). De cette façon il est possible d’inclure du code qui sera exécuté dans la chaîne. Tout ce qui est placé après un signe de dollar ($) et entre deux accolades ({}) sera exécuté. Par exemple, la chaîne let équation = `${2+2}=4`; deviendra c“4=4”.

Nombre

Un nombre peut être un chiffre, un entier négatif, un nombre décimal… En JavaScript, il sont dénotés sans symboles particuliers. Par exemple : let pi = 3.14;.

Attention! Les nombres décimaux s’écrivent à l’anglaise avec un point et non à la Française avec une virgule.

La valeur d’une variable de type nombre peut aussi être NaN, qui signifie Not A Number, ou Infinity si le nombre est trop gros. Mais nous verrons ces deux autres valeurs plus en détail dans le chapitre sur les opérations.

Valeur booléenne

C’est peut-être le type de variable qui semble le plus compliqué, et pourtant c’est certainement l’un des plus simples. Il a seulement deux valeurs possibles. true (vrai) et false (faux). Comme les nombres, ce type de valeur ne nécessite pas de symbole syntaxique particulier.

Par exemple : let assezÂgé = false;.

Fonction

Une fonction est une forme de variable qui est en quelque sorte un sous-programme. Elle contient une série d’instructions qui peut prendre des données et en retourner d’autres. Il y a trois façons principales de les dénoter.

Premièrement, il est possible d’utiliser le mot-clé function. Une déclaration avec cette syntaxe doit suivre le modèle suivant : 

												let variable = function(arg1, arg2, arg3... ) {
  //code
}

En voici un exemple plus concret : 

												let message = function(nom, âge) {
	const texte = `Tu t’appelles ${nom} et tu as ${âge} ans.`;
	alert(texte);
}

Deuxièmement, il est possible d’omettre la déclaration avec let et de nommer la fonction après le mot-clé function. Par exemple : 

												function message(nom, âge) {
	let texte = `Tu t’appelles ${nom} et tu as ${âge} ans.`;
	alert(texte);
}

Finalement, il est possible de déclarer une fonction en utilisant une flèche (=>). Cette méthode est la plus moderne et est de plus en plus courante. Elle est très similaire à la première forme, sauf que le mot-clé function est remplacé par (arg1, arg2... ) =>. Par exemple : 

												let message = (nom, âge) => {
	const texte = `Tu t’appelles ${nom} et tu as ${âge} ans.`;
	alert(texte);
}

Les trois syntaxes sont plus ou moins équivalentes et interchangeables. C’est une question de préférence personnelle. Dans les trois syntaxes, vous avez sûrement remarqué qu’il y a d’autres variables entre des parenthèses : (nom, âge). C’est ce qu’on appelle des arguments. Ce sont les données qui sont fournies à la fonction. À l’intérieur de cette fonction, ils peuvent être traités comme des variables (tu en apprendras plus au cours du prochain chapitre). Dans l’exemple il y en avait deux, toutefois il aurait tout aussi bien pu n’en avoir aucun ou plus de deux.

La variable message en tant que telle n’est pas vraiment utile. Toutefois, il est possible d’utiliser la fonction que contient la variable avec une syntaxe particulière. Dans mon exemple, si je voulais afficher un message disant « Tu t’appelles John Doe et tu as 16 ans. », j’écrirais message(“John Doe”, 16). Si la fonction n’avait pas eu d’arguments j’aurais simplement pu écrire message().

Les fonctions peuvent aussi retourner une donnée lorsqu’elles sont exécutées, avec le mot-clé return. Par exemple : 

													let message = (nom, âge) => {
	const texte = `Tu t’appelles ${nom} et tu as ${âge} ans.`;
	return texte;
}

Exécuter message(“John Doe”, 16) retourne la valeur “Tu t’appelles John Doe et tu as 16 ans.”. Il est possible d’assigner cette valeur à une autre variable ou de l’afficher, par exemple avec alert.

Il est à noter que return arrête complètement l'exécution de la fonction. Donc tout ce qui se trouve après cette instruction ne sera jamais exécuté.

Liste

Les listes ou Array en anglais sont une collection de variables. Elles peuvent contenir n’importe quel type de variables : des nombres, des chaînes de caractères, des fonctions, d’autres listes… Elles se notent avec des crochets : []). Par exemple : let paysVisités = [“Mexique”, “France”, “Norvège”];.

Pour accéder à un élément précis d’une liste, on utilise aussi les crochets. Par exemple : paysVisités[1]. Étrangement, cette instruction retourne la valeur “France”. C’est parce que la numérotation des listes commence à zéro et non à un.

Pour accéder à la longueur d’une liste, on utilise sa propriété length. Par exemple : paysVisités.length retourne 3. C’est normal puisqu’il y a trois éléments.

Donc, pour accéder au dernier élément d’une liste, il faut accéder à l’élément dont l’index est un de moins que la longueur de la liste. Par exemple : paysVisités[paysVisités.length - 1]; retourne “Norvège”.

Objet

Un objet, ou object en anglais, est très similaire à une liste. C’est aussi une collection de données. Toutefois, ces données ne sont pas classées selon un index numérique, mais plutôt avec des noms. De plus les objets sont délimités par des accolades ({}) et non des crochets ([]). À l'intérieur de l'objet, les données sont associées à un nom avec deux points (:) et sont séparées par des virgules. Par exemple : 

											let informations = {
	nom: “John Doe”,
	dateDeNaissance: “8 décembre 2019”,
	filmPréféré: “Pulp Fiction”
}
Attention! Il ne faut pas mettre de virgule après le dernier élément de l'objet.

Comme avec les listes, on peut utiliser les crochets ([]) pour accéder à un élément précis de l’objet. Cette fois-ci toutefois, on n’utilise pas un index, mais plutôt le nom, sous forme de chaîne de caractères1. Par exemple : informations[“filmPréféré”] retourne “Pulp Fiction”.

Il est aussi possible d’accéder à une valeur particulière avec un point. Par exemple, informations.nom retournera “John Doe”.
